# 기능 명세서: 사용자 인증 (회원가입/로그인)

이 문서는 사용자 인증 기능(회원가입, 로그인)의 프론트엔드 구현을 위한 명세와 설계 가이드를 제공합니다.

## 1. 핵심 원칙

-   **컴포넌트 기반 아키텍처 (CBA)**: 재사용 가능한 원자 단위의 UI 컴포넌트를 먼저 개발하고, 이를 조합하여 페이지를 구성합니다.
-   **관심사의 분리**: 각 컴포넌트와 로직은 아키텍처 원칙에 따라 명확하게 분리된 위치에 존재해야 합니다.
-   **도메인 주도 UI**: `apps/web`은 UI를 조립하는 껍데기(Shell)의 역할만 하며, 실제 UI 컴포넌트는 해당 기능이 속한 `domain` 패키지가 소유합니다.

---

## 2. 공통 컴포넌트 명세 (`01-core`)

프로젝트 전반에서 재사용될 기본 UI 컴포넌트입니다.

### 2.1. 버튼 컴포넌트 (`<Button />`)

-   **위치**: `01-core/src/browser/components/Button.tsx`
-   **목적**: 일관된 디자인의 버튼을 제공합니다.
-   **Props (속성)**:
    -   `children`: 버튼 내부에 표시될 텍스트 또는 아이콘.
    -   `onClick`: 버튼 클릭 시 실행될 함수.
    -   `type`: `button`, `submit` 등 버튼의 타입.
    -   `disabled`: (선택) 버튼 비활성화 여부.

### 2.2. 글자 입력 컴포넌트 (`<Input />`)

-   **위치**: `01-core/src/browser/components/Input.tsx`
-   **목적**: 유효성 검사 기능이 내장된 텍스트 입력 필드를 제공합니다.
-   **Props (속성)**:
    -   `type`: `text`, `email`, `password` 등 input 타입.
    -   `value`: 입력 값 (상태는 부모 컴포넌트에서 관리).
    -   `onChange`: 값이 변경될 때 실행될 함수.
    -   `placeholder`: (선택) 안내 문구.
    -   `validation`: (선택) 유효성 검사 규칙을 담은 객체.
        -   `required`: (선택) 필수 입력 여부.
        -   `maxLength`: (선택) 최대 글자 수.
        -   `pattern`: (선택) 정규표현식 패턴 (e.g., 이메일 형식).
        -   `errorMessage`: 유효성 검사 실패 시 툴팁으로 보여줄 메시지.
-   **동작**:
    -   `onChange` 이벤트 발생 시, `validation` prop에 명시된 규칙에 따라 값의 유효성을 검사합니다.
    -   유효성 검사는 `02-feature`에 구현된 공통 유효성 체크 로직을 호출하여 수행합니다.
    -   유효성 검사에 실패하면, `errorMessage`를 **툴팁(Tooltip)** 형태로 입력 필드 근처에 표시합니다.

---

## 3. 공통 유효성 검사 로직 (`02-feature`)

-   **위치**: `02-feature/src/validation_service.ts`
-   **목적**: 여러 도메인에서 재사용 가능한 유효성 검사 함수를 제공합니다.
-   **구현**:
    -   `validate(value, rules)` 형태의 함수를 구현합니다.
    -   `rules` 객체에는 `required`, `maxLength`, `pattern` 등이 포함될 수 있습니다.
    -   검사 결과(성공/실패)를 반환합니다. `<Input />` 컴포넌트는 이 결과를 받아 툴팁을 표시할지 결정합니다.

---

## 4. 기능 구현: 회원가입 (`03-domain/user`)

### 4.1. 회원가입 페이지 (`SignupPage`)

-   **위치**: `03-domain/user/browser/pages/SignupPage.tsx`
-   **목적**: 회원가입에 필요한 UI를 조합하여 하나의 페이지를 구성합니다.
-   **구현**:
    1.  `useState`를 사용하여 이메일, 이름, 비밀번호의 상태를 관리합니다.
    2.  `01-core`에서 만든 `<Input />` 컴포넌트를 여러 개 사용하여 각 입력 필드를 만듭니다. 각 `<Input />`에는 필요한 `validation` 규칙을 prop으로 전달합니다.
    3.  `01-core`에서 만든 `<Button />` 컴포넌트를 사용하여 "가입하기" 버튼을 만듭니다.
    4.  폼 제출(submit) 시 `signupUsecase`를 호출하여 실제 회원가입 로직을 실행합니다.

---

## 5. 기능 구현: 로그인 (`03-domain/user`)

-   회원가입과 동일한 방식으로 구현합니다.
-   `LoginPage.tsx`를 생성하고, `01-core`의 `<Input />`, `<Button />` 컴포넌트를 재사용하여 로그인 폼을 구성합니다.
-   로그인에 필요한 유효성 검사 규칙을 각 `<Input />`에 전달합니다.
