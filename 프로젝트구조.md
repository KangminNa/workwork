# React + NestJS 모노레포 구조

```
workwork/
├── package.json            # 워크스페이스/루트 스크립트
└── apps/
    ├── client/             # React + Vite 프론트엔드
    │   ├── index.html
    │   ├── src/
    │   │   ├── App.tsx     # API 결과를 렌더링하는 메인 컴포넌트
    │   │   └── main.tsx    # React DOM 진입점
    │   └── vite.config.ts  # dev 서버(5173) + Nest 프록시 설정
    └── server/             # NestJS 백엔드
        ├── src/
        │   ├── main.ts     # Nest 부트스트랩, CORS 허용
        │   ├── app.module.ts
        │   ├── app.controller.ts
        │   └── app.service.ts
        └── tsconfig*.json  # 빌드/개발용 TS 설정
```

## 설치 & 실행 순서

1. 루트에서 모든 의존성 설치
   ```bash
   npm install
   ```
2. 개발 서버 동시 실행 (React 5173 + Nest 3000)
   ```bash
   npm run dev
   ```
   - 백엔드만 필요하면 `npm run dev:server`
   - 프론트만 필요하면 `npm run dev:client`
3. 브라우저에서 http://localhost:5173 접속 → React 화면이 `/api/hello`를 통해 Nest의 "Hello from NestJS!" 메시지를 보여줍니다.

## 구성 설명
- **NestJS**: CORS 활성화 상태로 3000번 포트에서 JSON `{ message: 'Hello from NestJS!' }` 응답을 제공합니다.
- **React**: Vite dev 서버가 `/api/*` 요청을 Nest로 프록시하여, 환경변수 없이도 바로 연동 됩니다.
- **공통 워크플로우**: 루트 `package.json`의 `concurrently` 스크립트가 두 앱을 동시에 켜 주므로 `npm run dev`만 기억하면 됩니다.

Hello World 이후에는 Nest에 새로운 모듈을 추가하고, React에서 SWR/React Query 등으로 확장해 나가면 됩니다.
